# ansible-webserber-role
## Описание проекта
Автоматизированная Ansible-роль для развертывания полноценного веб-стека. Проект демонстрирует переход от линейных плейбуков к модульной структуре ролей, обеспечивая масштабируемость и безопасность конфигурации.
Этот репозиторий содержит Ansible-роль для автоматизированного развертывания веб-стека, включающего Nginx, Flask (через Gunicorn) и базу данных PostgreSQL. Проект демонстрирует навыки рефакторинга монолитных плейбуков в модульную структуру.

## Описание компонентов
### 1. Веб-сервер (Nginx)
Конфигурация Nginx настроена на работу в режиме Reverse Proxy:
1) Основной путь (/) обслуживает статические файлы из директории приложения.
2) Запросы к /api перенаправляются на бэкенд Flask, работающий на порту 5000.
3) Порт прослушивания и пути настраиваются через переменные nginx_port и app_path.

### 2. Бэкенд (Flask + Gunicorn)
Приложение Flask развертывается как системная служба (Systemd):
1) Используется виртуальное окружение Python.
2) Сервер запускается с 3 рабочими процессами для обработки запросов.
3) Настроена политика автоматического перезапуска службы при сбоях (Restart=always).

### 3. База данных (PostgreSQL)
Связь приложения с БД реализована через переменные окружения, передаваемые в Unit-файл сервиса:
1) Учетные данные подставляются из зашифрованного хранилища Ansible Vault.
2) Служба Flask настроена на запуск строго после инициализации сети и базы данных.

## Структура роли
```
.
├── roles/
│   └── webserver_role.yml           # Основной плейбук для запуска роли
└── webserver_role/                  # Директория самой роли
    ├── defaults/
    │   └── main.yml                 # Дефолтные (перезаписываемые) переменные
    ├── files/
    │   ├── app.py                   # Исходный код Flask-приложения 
    │   └── index.html               # Статическая страница фронтенда 
    ├── handlers/
    │   └── main.yml                 # Обработчики (restart nginx/flask)
    ├── tasks/
    │   └── main.yml                 # Основная логика установки и настройки
    ├── templates/
    │   ├── flask.service.j2         # Шаблон Systemd юнита 
    │   └── nginx.conf.j2            # Шаблон конфигурации Nginx 
    └── vars/
        ├── main.yml                 # Постоянные переменные проекта 
        └── vault.yml                # Зашифрованные секреты БД (пароли)

```
## Запуск
Для запуска роли на локальной машине используйте команду:
```bash
ansible-playbook webserver_role.yml --ask-vault-pass
```
